## 网络流量流向

### HTTP 请求流程

```
1. 用户浏览器
   ↓ (HTTP/HTTPS)
2. DNS 解析 → 负载均衡公网 IP
   ↓
3. 负载均衡 (SLB)
   ├─ SSL 终止 (HTTPS → HTTP)
   ├─ 健康检查
   └─ 选择后端服务器 (WRR 算法)
   ↓ (HTTP + X-Forwarded-For)
4. ECS Nginx (端口 80)
   ├─ 获取真实 IP
   ├─ 静态文件: 直接返回
   └─ API 请求: 代理到 Node.js
   ↓
5. Node.js Express (端口 3000)
   ├─ 路由处理
   ├─ 业务逻辑
   ├─ 数据库查询 → RDS
   └─ 文件上传 → OSS
   ↓
6. 响应返回
   Node.js → Nginx → SLB → 用户
```

### 健康检查流程

```
每 10 秒:
  负载均衡 → GET /health → ECS Nginx
  ↓
  Nginx 返回 200 "healthy"
  ↓
  ├─ 成功 3 次 → 标记为健康
  └─ 失败 3 次 → 标记为不健康，停止转发流量
```

## 扩展方案

### 当前配置（单 ECS）
```
负载均衡 → ECS #1 (100% 流量)
```

### 扩展到 2 台 ECS
```
负载均衡 → ECS #1 (权重 100, 50% 流量)
         → ECS #2 (权重 100, 50% 流量)
```

### 扩展到 N 台 ECS + 多可用区
```
负载均衡 (主可用区 A + 备可用区 B)
  ├─ 可用区 A
  │  ├─ ECS #1 (权重 100)
  │  └─ ECS #2 (权重 100)
  └─ 可用区 B
     ├─ ECS #3 (权重 100)
     └─ ECS #4 (权重 100)
```
